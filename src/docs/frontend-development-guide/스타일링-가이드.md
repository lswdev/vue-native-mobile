## 1. Tailwind CSS 규칙

### 기본 원칙

- **유틸리티 우선(Utility-First)** 원칙 준수
- 반복되는 클래스 조합은 `@apply`로 추출하거나 컴포넌트화
- 임의 값(`[...]`) 남용 금지 → `tailwind.config.ts`에서 디자인 토큰 정의

### tailwind.config.ts 필수 설정

```typescript
// tailwind.config.ts
import type { Config } from 'tailwindcss'

export default {
  content: ['./index.html', './src/**/*.{vue,ts,tsx}'],
  darkMode: 'class',
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: 'var(--color-primary)',
          50:  'var(--color-primary-50)',
          500: 'var(--color-primary-500)',
          700: 'var(--color-primary-700)',
        },
        surface: {
          DEFAULT: 'var(--color-surface)',
          raised: 'var(--color-surface-raised)',
        },
      },
      fontFamily: {
        sans: ['Pretendard', 'sans-serif'],
        mono: ['IBM Plex Mono', 'monospace'],
      },
      spacing: {
        '18': '4.5rem',
        '88': '22rem',
      },
      borderRadius: {
        'card': '0.75rem',
      },
      boxShadow: {
        'card': '0 2px 12px 0 rgba(0,0,0,0.08)',
        'popup': '0 8px 32px 0 rgba(0,0,0,0.16)',
      },
    },
  },
  plugins: [],
} satisfies Config
```

### 클래스 작성 순서 (권장)

```
레이아웃 → 크기 → 간격 → 배경/테두리 → 타이포그래피 → 효과 → 반응형 → 상태
```

```html
<!-- ✅ 올바른 순서 예시 -->
<div class="
  flex items-center justify-between
  w-full h-14
  px-4 py-2
  bg-surface border border-gray-200 rounded-card
  text-sm font-medium text-gray-900
  shadow-card
  md:h-16
  hover:bg-surface-raised
">
```

### 반응형 브레이크포인트

```
sm:  640px   모바일 가로
md:  768px   태블릿
lg:  1024px  소형 데스크탑
xl:  1280px  데스크탑
2xl: 1536px  대형 모니터
```

- **모바일 퍼스트** 원칙으로 작성
- 단, 본 프로젝트(KIE-REMS)는 대시보드 중심이므로 `lg:` 기준 기본 레이아웃 설계

### Tailwind와 BEM 병행

| 용도 | 권장 방식 |
|------|-----------|
| 레이아웃·간격·반응형·색 토큰 등 유틸리티 | Tailwind 클래스 |
| 컴포넌트 구조·상태·도메인 전용 스타일 | BEM 클래스 (`<style scoped>` / CSS Modules) |

동일 엘리먼트에 Tailwind와 BEM을 함께 쓸 수 있다. BEM은 **의미·구조**, Tailwind는 **반복 유틸**에 집중한다.

```html
<!-- ✅ Tailwind(유틸) + BEM(구조·상태) -->
<aside
  class="gis-map-panel flex h-full flex-col"
  :class="{ 'gis-map-panel--collapsed': isCollapsed }"
>
  <header class="gis-map-panel__header px-4 py-2">...</header>
</aside>
```

## 2. CSS / CSS Modules 규칙

### BEM 네이밍 규칙

본 프로젝트의 **커스텀 CSS 클래스**는 BEM을 표준으로 한다. 적용 대상은 `<style scoped>`, CSS Modules(`.module.css`), `styles/` 전역 컴포넌트 클래스(`@layer components`)이다.

#### Block · Element · Modifier

```
.block { }                        /* 독립 컴포넌트 루트 */
.block__element { }               /* Block 내부 요소 */
.block--modifier { }              /* Block 상태·변형 */
.block__element--modifier { }     /* Element 상태·변형 */
```

| 규칙 | 설명 |
|------|------|
| Block | 재사용 가능한 UI 단위의 **루트**. Vue 컴포넌트 1개당 Block 1개를 원칙으로 한다. |
| Element | Block에만 속하는 하위 부분. **`__`(언더스코어 2개)** 로 연결한다. |
| Modifier | 외형·상태·모드 변화. **`--`(하이픈 2개)** 로 연결한다. |
| 이름 형식 | 전부 **kebab-case** (예: `energy-chart`, `facility-detail`) |

#### Block 이름 짓기

- Vue 컴포넌트 `GisMapPanel.vue` → Block `gis-map-panel` (PascalCase → kebab-case)
- 페이지 전용 Block은 페이지·도메인 접두를 붙인다. (예: `gis-facility-detail`, `auth-login-form`)
- Block 이름은 **짧고 도메인 맥락이 드러나게** 짓는다. (`panel`, `wrapper` 단독 사용 금지)

#### Element 규칙

- Element는 **항상 Block 이름으로 시작**한다. (`__header`, `__body` — Block 생략 금지)
- Element 중첩 표기(`block__a__b`)는 **사용하지 않는다**. 하위 구조가 깊어지면 Element 이름을 구체화하거나 Block을 분리한다.

```css
/* ❌ Element 중첩 금지 */
.gis-map-panel__legend__item { }

/* ✅ Element 이름으로 의미 표현 */
.gis-map-panel__legend-item { }
```

#### Modifier 규칙

- Modifier는 **상태·테마·크기·열림/닫힘** 등 변형에만 사용한다.
- 불리언 상태는 `--active`, `--disabled`, `--open`, `--collapsed`처럼 **형용사/과거분사**로 통일한다.
- Modifier만 단독 클래스로 쓰지 않는다. (항상 Block 또는 Element와 함께)

```html
<!-- ✅ -->
<div class="base-button base-button--danger">...</div>
<button class="base-button__icon base-button__icon--spinning">...</button>

<!-- ❌ Modifier 단독 -->
<div class="--danger">...</div>
```

#### Vue SFC에서의 적용

```vue
<script setup lang="ts">
const isCollapsed = ref(false)
</script>

<template>
  <section
    class="gis-map-panel"
    :class="{ 'gis-map-panel--collapsed': isCollapsed }"
  >
    <header class="gis-map-panel__header">
      <h2 class="gis-map-panel__title">GIS 지도</h2>
    </header>
    <div class="gis-map-panel__body">
      <slot />
    </div>
  </section>
</template>

<style scoped>
.gis-map-panel {
  display: flex;
  flex-direction: column;
  height: 100%;
}

.gis-map-panel--collapsed {
  width: var(--sidebar-collapsed-width);
}

.gis-map-panel__header {
  flex-shrink: 0;
  height: var(--header-height);
}

.gis-map-panel__title {
  font-size: 0.875rem;
  font-weight: 600;
}
</style>
```

- `<style scoped>` 사용 시에도 **클래스 이름은 BEM**을 유지한다. Vue가 scope attribute를 붙이므로 전역 충돌은 줄어든다.
- `:deep()` / `:slotted()` 사용 시에도 **대상 선택자는 BEM 이름**을 유지한다.

```vue
<style scoped>
.gis-map-panel :deep(.gis-map-panel__chart) {
  min-height: 12rem;
}
</style>
```

#### CSS Modules와 BEM

CSS Modules 파일의 **클래스 선택자 문자열**은 BEM을 따른다. TS에서 참조하는 **export 키**는 camelCase로 매핑할 수 있다.

```vue
<script setup lang="ts">
import styles from './gisMapPanel.module.css'
</script>

<template>
  <div :class="styles['gis-map-panel']">
    <div :class="styles['gis-map-panel__header']">...</div>
  </div>
</template>
```

```css
/* gisMapPanel.module.css — 선택자는 BEM */
.gis-map-panel {
  position: relative;
  overflow: hidden;
}

.gis-map-panel__header {
  display: flex;
  align-items: center;
  height: var(--header-height);
}

.gis-map-panel__header--dark {
  background-color: #1e293b;
}
```

#### 전역 `@layer components`와 BEM

디자인 시스템 수준의 공통 클래스도 BEM을 적용한다.

```css
@layer components {
  .btn { /* Block */
    @apply inline-flex items-center gap-2 rounded-lg font-medium text-sm
           transition-colors duration-200
           disabled:cursor-not-allowed disabled:opacity-50;
  }

  .btn--primary {
    @apply bg-primary px-4 py-2 text-white hover:bg-primary-700;
  }

  .card { /* Block */
    @apply rounded-card bg-surface p-4 shadow-card;
  }

  .card--elevated {
    @apply shadow-popup;
  }
}
```

#### BEM 안티패턴 (금지)

| 금지 | 이유 | 대안 |
|------|------|------|
| `header`, `content`, `wrapper` 등 단독 일반명 | 전역·scoped 모두 충돌 위험 | `gis-map-panel__header` |
| `block__a__b` Element 중첩 | BEM 표준 위반, 유지보수 어려움 | `block__legend-item` |
| 태그·ID 선택자로 구조 스타일링 (`.panel div span`) | 마크업 결합·특이도 증가 | Element 클래스 |
| Modifier 단독 (`.--active`) | Block 맥락 없음 | `.block--active` |
| Block/Element에 camelCase (`gisMapPanel`) | 프로젝트 CSS 규칙 불일치 | `gis-map-panel` |

### 전역 CSS 구조 (`styles/main.css`)

```css
/* styles/main.css */

/* 1. Tailwind Directives */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* 2. CSS Custom Properties (디자인 토큰) */
@import './variables.css';

/* 3. 기본 타이포그래피 */
@layer base {
  html {
    font-family: 'Pretendard', sans-serif;
    -webkit-font-smoothing: antialiased;
  }
}

/* 4. 공통 컴포넌트 클래스 (BEM) */
@layer components {
  .btn {
    @apply inline-flex items-center gap-2 rounded-lg font-medium text-sm
           transition-colors duration-200
           disabled:opacity-50 disabled:cursor-not-allowed;
  }

  .btn--primary {
    @apply bg-primary px-4 py-2 text-white hover:bg-primary-700;
  }

  .card {
    @apply bg-surface rounded-card shadow-card p-4;
  }
}
```

### CSS Modules 사용 기준

- 전역 영향을 줄 수 있는 복잡한 컴포넌트에서만 CSS Modules 사용
- 일반 컴포넌트는 `<style scoped>` 우선

```vue
<script setup lang="ts">
import styles from './gisMapPanel.module.css'
</script>

<template>
  <div :class="styles['gis-map-panel']">
    <div
      :class="[
        styles['gis-map-panel__header'],
        styles['gis-map-panel__header--dark'],
      ]"
    >
      ...
    </div>
  </div>
</template>
```

```css
/* gisMapPanel.module.css — 클래스 선택자는 BEM */
.gis-map-panel {
  position: relative;
  overflow: hidden;
}

.gis-map-panel__header {
  display: flex;
  align-items: center;
  height: var(--header-height);
}

.gis-map-panel__header--dark {
  background-color: #1e293b;
}
```
